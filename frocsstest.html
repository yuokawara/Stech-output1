Objectのなかに

  • Component
  • Project

詳細度がより高くなる。使いこなせると、どの設計よりも便利に活用できる

ただし、学習コストは高い

Foundation

土台になる部分、Baseのカテゴリと同じ。最初に読み込まれる部分。

MCSSのFoundationと同じと思うとわかりやすい

Layout

ヘッダーやフッターなどサイト全体のブロックを定義する

Object

サイト内で繰り返されるパターンの定義 さらに3つに分類される

ここが特徴的

Component

再利用可能の一番小さいモジュール。ボタン、ページネーション等

Project

MCSSのProjectと同じ。ユーザープロフィールや画像ギャラリー

具体的な要素。

Utility

ComponentやProjectレイヤーのModifierで解決が難しい、わずかなスタイル調整のためのクラス。

名前の通り便利クラス。MCSSのCosmeticに近い。

Componentには.c-,Projectには.p-,Utilityには.u-の命名規則をつける。
